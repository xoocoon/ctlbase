message module

Classes for capturing messages from controls and other sources like REST APIs.

Messages can be appended to an instance of MessageBuffer before actually writing them to the console and/or a log file. An individual message is captured with multiple attributes, not just with its human-readable text. This has two benefits:

  • Messages can be translated into another language in one go and on-demand. See addTranslation().

  • Message semantics can be queried programatically, e.g. by checking for a certain message type via hasKey(). Another example is checking for a certain message severity like ERROR via hasSeverity().

class ctlbase.message.Message(textEn, key, arguments=(), severity: Severity = Severity.NORMAL, timestamp_iso=None, ns=None, correlationId=None)

A message captured with several attributes.

textEn: str

The human-readable message text in English (en). It serves multiple purposes:

  • At runtime: If the message is to be shown to a user whose preferred language is en, the message text is used “as is”.

  • If the user has a different preferred language, the English message text is used as a fallback if no appropriate translation is available.

  • At development time: If the English message text occurs as a literal str in the source code, it serves as an additional documentation of the source code, e.g. a literal error message for a specific case.

  • Also, it serves as the primary reference when translating the message text into other languages.

key: str
An alphanumeric key for the message type. It is used to check for messages that have occurred, as well as a key into translation files. It must be unique within the message namespace (ns).
Example: FILE_NOT_FOUND
arguments: list | tuple | dict

The message arguments, i.e. language-neutral literals that can be embedded in the message texts of various languages. Examples:

  • an int like 5 for the number of failed retries

  • a str representing a user-supplied value, e.g. /tmp/command.log for a log file path

  • a float like 1.532 representing the elapsed time in seconds

If the arguments are specified as a dict instead of an ordered list or tuple, the dict keys must be semantic identifiers for the arguments. Examples:

  • retryCount for the number of failed retries

  • log for a log file path

  • elapsed for the elapsed time

severity: Severity

The message severity.

timestamp_iso: str | datetime.datetime
An optional timestamp in ISO 8601 format or an instance of datetime.datetime directly. It should represent the point in time when the message occurred originally.
Example: 2024-08-01T14:38:32.499588
ns: str
An alphanumeric identifier for the message namespace within which all message keys must be unique. It usually represents the application or domain from which the message originates.
Examples: HOME_THEATER or ENGINE_CONTROL
correlationId: str

An optional correlation ID, i.e. an alphanumeric identifier used to operationally group several messages. As an example, it can be an identifier for a batch command that produces several messages across several message namespaces.

Compares this message to another object by all attributes except the str values of the message texts.

class ctlbase.message.MessageBuffer(nsDefault, jsonIndentDefault=4, lngDefault='en', isColoringDefault=None)

A buffer for messages.

Messages can be appended without writing them to the console directly. Appended messages can be queried programatically until the buffer is cleared via clear(). For instance, to check for a certain message severity like ERROR, hasSeverity() is used.

To eventually print the messages to the console or another IO stream, the printText() method is used.

The COLOR_PRINT environment variable determines the default behavior for coloring in text output functions if isColoringDefault was initialized as None. The allowed values for the environment variable are true and false.

nsDefault: str

The default namespace to be assigned to messages without a namespace of their own.

jsonIndentDefault: int

The default number of spaces used to indent one hierarchy level in JSON output functions, namely getJson().

lngDefault: str

The two-letter language code to be used as the default in text output functions, namely printText() and getText().

isColoringDefault

If True, colored text is the default in text output functions, namely printText() and getText(). Overrides the COLOR_PRINT environment variable. Defaults to True if the environment variable is unset.

clear()

Clears all messages in the buffer.

The default values of the buffer are retained.

append(msg, key=None, arguments=(), severity: Severity = Severity.NORMAL, timestamp_iso=None, ns=None, correlationId=None, isDeduplicate=True)

Appends a message to the buffer. The message may be given in one of various forms:

  1. As an instance of Message, provided in the msg parameter. Example:

    msg = Message("Configuration file not found", 'CONFIG_NOT_FOUND', severity=Severity.ERROR)
msgBuffer.append(msg)

  2. As individual attributes, of which the English message text (textEn) must be provided in the msg parameter. What is more, key must be provided. All other attributes are optional and assume default values if not provided. Example:

    msgBuffer.append("Configuration file not found", 'CONFIG_NOT_FOUND', severity=Severity.ERROR)

  3. As an instance of ErrorMessage, provided in the msg parameter. Example:

    msgError = ErrorMessage("Configuration file not found", 'CONFIG_NOT_FOUND', exitCode=1)
msgBuffer.append(msgError)

Regardless of the chosen form, if the message namespace (ns) is None, the value of nsDefault is used.

Parameters:

  • msg¶ (Message | ErrorMessage | str) – The message to appemd, depending on the chosen form (see above).

  • key¶ (str) – The message key (key), if form 2 is chosen.

  • arguments¶ (list | tuple | dict) – The message arguments (arguments), if form 2 is chosen.

  • severity¶ (Severity) – The message severity (severity), if form 2 is chosen.

  • timestamp_iso¶ (str) – A timestamp (timestamp_iso) if form 2 is chosen.

  • ns¶ (str) – The message namespace (ns), if form 2 is chosen.

  • correlationId¶ (str) – A correlation ID (correlationId), if form 2 is chosen.

  • isDeduplicate¶ (bool) – If True, appends the message only if it is not yet contained in the buffer, according to __eq__().

getExitCode()

Determines the most appropriate numeric exit code for the process, based on the messages in the buffer.

If instances of ErrorMessage were appended to this buffer, the exitCode of the instance appended last is returned.
Otherwise, the exit code depends on the message in the buffer with the “highest” severity:
4 if at least one message has the severity ERROR.
3 if at least one message has the severity WARNING.
0 otherwise.
Returns:

The numeric exit code.

addTranslation(lng, isOkMissingFile=False, isOkMissingKey=True, isOkMissingArgument=False, correlationId=None)

Loads a translation file and adds the corresponding translations to the buffer.

The syntax of translation files is a basic version of i18next. Message arguments can either be interpolated by index or by semantic key:

"Freigabe {{0}} ausgehängt."
# or
"Freigabe {{share}} ausgehängt."

In the first case, arguments must be of type list or tuple. In the second case, they must be of type dict. Default values for arguments may be specified in the translation file in a top-level object named defaultVariables. It must contain key/value pairs of argument indices or keys and the corresponding default values.

The search path for a translation file is determined by TRANSLATION_LOCATIONS.

Parameters:

  • lng¶ (str) – Two-letter language code for the translations to be added, e.g. de.

  • isOkMissingFile¶ (bool) – If False, a warning with key TRANSLATION_FILE_NOT_FOUND is appended to the buffer if a translation file for lng cannot be found.

  • isOkMissingKey¶ (bool) – If False, a warning with key TRANSLATION_KEY_NOT_FOUND is appended to the buffer if the key of a message in the buffer cannot be found in the translation file.

  • isOkMissingArgument¶ (bool) – If False, a warning with key MESSAGE_ARGUMENT_NOT_FOUND is appended to the buffer if the translation file requires an argument that cannot be found in the message with the corresponding key.

  • correlationId¶ (str | None) – The correlation ID under which possibly occurring warning messages are appended to the buffer.

class ctlbase.message.LoggingMessageBuffer(loop, nsDefault, jsonIndentDefault=4, lngDefault='en', isColoringDefault=None, logPath=None, firstColumnWidth=24, isLogImmediately=True, isLogToConsole=False)

Bases: MessageBuffer

A message buffer capable of logging messages to a file.

logPath: str

The log file path. It is opened only when openLog() is called.

firstColumnFormatStr: str

The printf-style string used to format the first column in the log output.

The first column contains the timestamp of the corresponding log entry, right-padded with spaces to match the number of characters given in firstColumnWidth.

isLogImmediately: bool

If True, any message is written to the log file as soon as it is appended to the buffer. If False, all the messages in the buffer are written to the log file when the buffer is closed via its close() method.

isLogToConsole: bool

If True, messages are additionally written to stdout at the same time when they are written to the log file.

  • firstColumnWidth¶ (int) – Sets the value of firstColumnFormatStr. If None, the existing value of firstColumnFormatStr is used.

append(msg, key=None, arguments=(), severity: Severity = Severity.NORMAL, timestamp_iso=None, ns=None, correlationId=None)

Immediately writes a message to the log file if isLogImmediately is True and a a log file has been opened via openLog(). In this case, messages are not buffered. Hence, they cannot be queried programatically via hasKey() or hasSeverity() or any other method.

class ctlbase.message.MessageIterator(messages, iteratorType=Format.OBJECT, startIndex=0, isImmutable=False, argumentFilter=None, severityFilter=None, correlationIdFilter=None, lngForText='en')

Allows for (filtered) iteration over the messages in a buffer.

While iterating, messages may still be appended to the buffer.

class Format(value)

The format of messages to be yielded.

OBJECT = 0

An instance of Message.

TEXT = 2

The message text as a str.

TEXT_COLORED = 3

The message text as a str enclosed in ANSI control characters coloring the text according to severity.

__init__(messages, iteratorType=Format.OBJECT, startIndex=0, isImmutable=False, argumentFilter=None, severityFilter=None, correlationIdFilter=None, lngForText='en')
Parameters:

  • messages¶ (MessageBuffer | list[Message] | tuple[Message]) – The message sequence to iterate over.

  • iteratorType¶ (Format) – The format of the messages to be yielded.

  • startIndex¶ (int) – The (zero-based) index of the first message to yield. It must be less than len(messages). As an example, if startIndex is 2, iteration starts with the third message in messages.

  • isImmutable¶ (bool) – If True, the iterator uses an immutable copy of the message sequence. In this case, messages appended afterwards are not reflected in the iterator.

  • argumentFilter¶ (object | dict) –

    An optional filter for message arguments (arguments). The filter is specified as a scalar value, e.g. a str, an int or a bool. For a message to pass the filter, one of its arguments must exactly match the value.
    To match a key/value pair instead of just a value, the filter may be specified as a dict with a single entry. As an example, argumentFilter={'retryCount':5} matches message arguments specified as a dict, containing a semantic key retryCount with a value of 5.
    If None, messages are not filtered by arguments.

  • severityFilter¶ (Iterable) –

    An optional filter for message severities (severity). The filter is specified as an Iterable with one or more severities. For a message to pass the filter, its severity must match one of the given severities.
    If None, messages are not filtered by severity.

  • correlationIdFilter¶ (str) –

    An optional filter for correlation IDs (correlationId). For a message to pass the filter, its correlation ID must exactly match the given value.
    If None, messages are not filtered by correlation ID.

  • lngForText¶ (str) –

    Two-letter language code for the message texts to be returned.
    Defaults to en, if a corressponding translation cannot be found.
    Only relevant for formats TEXT and TEXT_COLORED.

Raises:

  • ValueError – If startIndex is invalid.

  • ValueError – If argumentFilter is an invalid dict.

class ctlbase.message.Severity(value)

The severity of a single message.

The member values denote the ANSI color codes for corresponding messages written to the console and/or a log file.

DEBUG = 90

Technical message intended for debugging.

ERROR = 91

Error message, e.g. for a failed action.

NORMAL = 39

Informative message with no special coloring.

SUCCESS = 92

Success message, e.g. for a completed action.

VERBOSE = 37

Verbose message providing more detail than NORMAL.

WARNING = 93

Warning message, e.g. for an action that completed, yet in an unexpected way.
